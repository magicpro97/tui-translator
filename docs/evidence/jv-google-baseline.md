## Decision

The live Google Translation baseline was not executed. This is an intentional
non-billable block, not a silent skip: the current repository can validate the
LF-04-v2 pending benchmark artifact, but it still lacks the two pieces required
to produce comparable live Google evidence.

| Blocker | Evidence | Impact |
|---|---|---|
| Full 300-row corpus not assembled | JV-02 committed a 20-row synthetic seed and a plan; the full corpus is generated by later tooling at evaluation time | Running Google over only the seed would not satisfy the 10x300 baseline requirement |
| Live Google mode not implemented | `mt_bench --with-google` performs cost preflight, then exits with `--with-google is not yet implemented` | No Google request path, per-round writer, or scorer exists yet |
| Requested `$3.00` cap is below conservative full-corpus estimate | `300 * 10 * 60 * $0.00002 = $3.60` | A full run would need actual corpus character-count preflight or a raised cap |

No Google network request was sent and no key value was printed or stored.

## Required next implementation

JV-04 remains blocked until a follow-up implementation provides:

1. A corpus builder/downloader that materializes the approved 300-row corpus
   from the JV-02 source plan into a local, hashed `corpus.jsonl`.
2. A live Google request path in `mt_bench` that reads a key without logging it,
   applies the cost cap before every billable request, writes per-round NDJSON,
   and rejects auth/4xx without retry.
3. Actual character-count cost preflight from the materialized corpus, or an
   explicit cap higher than the conservative `$3.60` estimate.
4. chrF++/BLEU scoring against `vi_refs` and redaction scans over all emitted
   artifacts.

## Privacy and redaction policy

This blocked artifact records key presence as a boolean only. It does not store
the configured key, request URLs, response bodies, source text, or translated
text. The sidecar below contains no per-round source/target records because no
billable rounds ran.
