Issue #224 evidence - JSONL session log schema

Scope implemented:
- Added `src/session/mod.rs` with versioned `SessionLogRecord`, `SessionHeader`,
  and `TranscriptSegment` serde types.
- Added schema versioning rules in the module documentation and a runtime
  version gate for future migrations.
- Added `tests/fixtures/session_log_v1.jsonl` as a concrete header + transcript
  fixture.
- Added `tests/session_schema.rs` to prove the fixture deserializes and optional
  STT confidence can be omitted.

Acceptance coverage:
- Timestamp, source text, target text, providers, languages, latency, cumulative
  audio/character counters, and estimated cost are represented in
  `TranscriptSegment`.
- Every record includes `schema_version`.
- The fixture validates through `serde_json` in the test suite.

Validation commands:
- `cargo fmt --all -- --check`
- `cargo test --test session_schema -- --nocapture`
- `WHISPER_DONT_GENERATE_BINDINGS=1 cargo clippy --all-targets --all-features -- -D warnings`
