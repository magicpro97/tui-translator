ACCEPTANCE CRITERIA MAP

AC1: README names recommended default model tier.
  Section: "CPU-only / offline mode > Recommended model tier"
  Evidence: Table row with `faster-whisper-base` marked **Recommended default**.
  Quote: "| **`faster-whisper-base`** | 139 MB | ~288 MB | 0.0% | **Recommended default** |"

AC2: README warns that CPU-only quality/latency varies by hardware.
  Section: "CPU-only / offline mode > Recommended model tier"
  Evidence: Explicit warning before the model table.
  Quote: "Quality and latency vary by hardware — always verify on your target machine."
  Also in "Known limitations": "The benchmark numbers above come from a single high-end host.
  A slower CPU will have higher real-time factors (RTF)."

AC3: Docs do not imply host Zoom privileges are needed.
  Section: "CPU-only / offline mode > When to use offline mode"
  Evidence: Explicit statement that no host account or privileges are required.
  Quote: "No Zoom host account or host privileges are needed."

8 GB LAPTOP USER RECOMMENDATION TEST
--------------------------------------
Section "Recommended model tier" explicitly states:
  "8 GB machines: Start with `faster-whisper-base`."
  "Do not use `small` on an 8 GB machine unless a co-run benchmark confirms
  RTF < 1.0 and sufficient headroom."
Model table shows base peak RSS ~288 MB, well within safe bounds.
Conclusion: 8 GB user finds a safe, clearly labeled recommendation.
